Kotlin

Kotlin의 소개

  • 구글의 함수형 언어인 코틀린을 통해서 안드로이드를 개발할 수 있도록 지원한다.
  • 장점
    – 간결함
    – 안전함
    – 다재다능함
    – 호환성
  • 비약적으로 코드의 양이 줄어들고, Null-Pointer 예외에 대한 안전성이 증가했다.
  • 함수형 언어가 비슷하기 때문에 코틀린을 배우면 파이썬,Swift, 자바스크립트로 금방 넘어갈 수 있다.

변수와 상수

  • 데이터를 담을 수 있는 그릇을 변수 또는 상수라고 한다.

변수란?

  1. 특징
  • 값을 담을 수 있는 그릇
  • 그릇에 값을 담고 비우고 다른 값을 담을 수 있다. 하지만, 그릇의 형식은 바꿀 수가 없다.
  • 값을 다른 값으로 바꿀 수 있다.
  • 초기값이 필요하지 않다.
  • var을 통한 선언
  • 자주 변환되는 데이터에 사용됨
  1. 종류

  • 숫자(Number)
    – Byte(8bit)
    -128 ~ 127
    – Short(16bit)
    -32,768 ~ 32,767
    – Int(32bit)
    -2147483648 ~ 2147483648
    – Long(64bit)
    -21억, 21억 이상인 수
    – Float(32bit)
    소수점 7자리까지 표현 가능
    – Double(64bit)
    소수점 14자리까지 표현 가능

  • 글자(Character)
    – 글자 한개만 담음
    – Char를 입력해서 선언
    – '(작은 따옴표)를 이용해서 입력

  • 논리값(Boolean)
    – true, false 값만 담을 수가 있음
    – Boolean 입력해서 선언

  • 배열(Array)
    – 같은 형식의 데이터가 반복적으로 쓰이는 것을 담을 때 사용
    – 2차원 데이터
    – Array<데이터 형식> 으로 선언
    – 데이터 형식 : 숫자, 글자, 문자열, 논리값, 사용자 정의 데이터 형식

  • 문자열(String)
    여러 개의 문자를 담을 수 있음
    String을 입력해서 선언
    "(큰 따옴표)를 이용해서 입력

상수란??

  1. 특징
  • 값을 담을 수 있는 그릇
  • 값을 바꿀 수 없다.
  • 초기값이 필요하다.
  • val을 통한 선언
  • 변환하면 안되는 중요한 데이터에 사용함

연산자

  1. 대입
  • (데이터를 받은 변수) = (데이터를 입력한 값)
  • 같다의 의미가 아님
  • 참고 : 수학에서 쓰는 '같다’의 연산자는 ==
  1. 산술 연산자
  • 더하기(+)
  • 빼기(-)
  • 곱하기(*)
  • 나누기(/)
  • 나머지(%)
  • 간단한 내용이므로 실습으로 익히자!
  1. 비교 연산자
  • 두 변수를 비교!
  • >
  • <
  • >=
  • <=
  • == (equal)
  • != (not)
  1. 범위 연산자
  • 자바에서는 없는 기능이고, 코틀린에 새로 도입된 기능!
  • (비교변수) in 시작값 … 마지막값
    비교 변수 값이 시작값과 마지막값 사이에 있는 지 확인
    z in 1…10
    1 <= z <= 10
    z !in 1…10
    1 >=z , z>=10 // 오히려 범위에 소속되지 않는 값을 물어봄

제어문

  1. if문
  • 조건을 통해서 분기를 함
  1. when문
  • java의 switch문과 같음
  • if - else 문이 많아서 코드의 가독성이 떨어질 경우 when문 사용!
  1. for문
  • 반복문
  • Range for문
    x값이 특정 범위 안에 있을 때 실행
  • Collection for문 두 가지 종류 존재
    가지고 있는 item만 접근해서 출력하고 종료!
    순차적으로 읽고 더 이상 읽을 내용이 없을 때 종료
  1. while문
  • for문과 비슷하지만, 조건이 들어가게 되며 조건이 참인 동안은 무한히 반복하게 된다.
  • 빠져나오기 위해서는 조건을 거짓으로 만드는 문장이 존재해야 한다.

함수(Functions)

  • Input이 들어오면 Output이 존재하는 구성

  • fun 함수이름 (입력값) : 반환값 {

    // 프로세스 처리

    // return 결과
    }

NullSafety - NullPointerException

  • 변수에 어떠한 값도 할당하지 않았을 경우 null이라고 하는데, 이 변수를 Computer가 읽게 된다면 null 깂을 읽게 되므로 심각한 오류를 발생시킨다.
  • 코틀린에서는 Null을 원천적으로 차단할 수 있는 NullSafety라는 개념을 도입하게 되었다.
  1. Null Safety 선언
  1. Null Safety 강제 해재
  • Compiler는 Null Safety에 의해서 보호를 받고 있는 변수에 접근할 수 없으므로 접근하기 위해서는 강제로 해제해주어야 한다.
  • 방법
    – 강제 해제 :
    (1) 방법 : !!를 사용하여 강제 but, 강제 해제를 많이 사용할 경우 오류가 발생하여 종료될 수 있기 때문에 주의해야 한다.
    (2) 사용하는 이유 : Null Safety로 선언된 변수는 Null Safety로 선언된 변수에만 담길 수 있기 때문

– 안전 해제 :
(1) 방법1 : !!를 사용하지 않고 자동 해제 되는 것
(2) 방법2 : ?를 사용하는 이유는 변수에 소속된 function이나 하위 클래스에 접근하고 할 때 사용한다.
