Java :: 2) 연산자

2018-08-03
Java

연산자와 관련된 내용 중에서 헷갈리거나 어려운 내용을 공부하고 정리했습니다.

연산자

  • x<<2+1 : 쉬프트 연산자(<<)는 덧셈 연산자보다 우선순위가 낮습니다. 그래서 해당 식은 x<<(2+1)과 같습니다.

  • data & 0xFF == 0 : 논리연산자(&)는 비교연산자(==)보다 우선순위가 낮으므로 비교연산을 먼저 수행한 후에 논리연산이 수행됩니다. 따라서 해당 식은 data & (0xFF == 0)과 같습니다.

  • x<-1 || x>3 && x<5 : 논리연산자 중에서 AND를 의미하는 &&가 OR을 의미하는 ||보다 우선순위가 높습니다. 그래서 해당 식은 x<-1 || (x>3 && x<5)과 같습니다.

  • 증감 연산자의 연산 순서 : 증감 연산자는 피연산자의 어느 쪽에 위치하는가에 따라 연산의 순서가 달라집니다. 다음 예제를 통하여 확인해볼 수 있습니다.

  • instanceof 연산자 : 참조 변수가 참조하고 있는 인스턴스의 실제 타입을 반환해줍니다. 즉, 해당 객체가 어떤 클래스나 인터페이스로부터 생성되었는지를 판별해 주는 역할을 합니다. instanceof 연산자는 왼쪽 피연산자인 인스턴스가 오른쪽 피연산자인 클래스나 인터페이스로부터 생성되었으면 true를 반환하고, 그렇지 않으면 false를 반환합니다.

산술 변환 : 산술 변환이란 연산을 수행하기 전에 피연산자 타입의 일치를 위해 자동 형 변환되는 것을 말합니다. 이 변환은 이항 연산에서 뿐만 아니라 단항 연산에서도 일어납니다. 규칙은 다음과 같습니다.

  1. 두 피연산자의 타입을 같게 일치시킨다. (보다 큰 타입으로 일치)

    • long + int -> long + long -> long
    • float + int -> float + float -> float

  2. 피연산자의 타입이 int보다 작은 타입이면 int로 변환됩니다.

    • byte + short -> int + int -> int
    • char + short -> int + int -> int

첫 번째 규칙은 자동 형변환처럼 피연산자의 값 손실을 최소화하기 위한 것이고, 두 번째 규칙은 정수형의 기본 타입인 int가 가장 효율적으로 처리할 수 있는 타입이기 때문에, 그리고 int보다 작은 타입, 예를 들면 char나 short의 표현범위가 좁아서 연산 중에 오버플로우(Overflow)가 발생할 가능성이 높기 때문에 만들어진 것입니다.

위의 예제에서 '1000000 * 1000000’의 결과가 1000000000000(2 * 10의 12승)임에도 불구하고, -727379968라는 결과가 출력되었습니다. 그 이유는 int 타입과 int 타입의 연산 결과는 int 타입인데, 연산결과가 int 타입의 최대 값인 1000000000(2 * 10의 9승)을 넘으므로 오버플로우(Overflow)가 발생했기 때문입니다. 이미 오버플로우가 발생한 값을 아무리 long 타입의 변수에 저장해도 소용이 없습니다.

b에서 처럼 1000000L으로 형 변환을 통해서 1000000도 long형으로 타입을 맞춘 후에 연산을 한 값을 long형 타입의 변수 b에 저장한다면 올바른 결과를 얻을 수 있습니다.

위 예제에서 c2++; 대신에 c2 = c2+1;을 사용하면 에러가 발생하게 됩니다. 이유는 c2+1의 연산결과는 int형이며, 그 결과를 다시 c2에 담으려면 형변환 연산자를 이용해 char형으로 형변환해야 하기 때문입니다.

‘a’+1은 리터럴간의 연산이기 때문에 에러가 발생하지 않습니다. 상수 또는 리터럴 간의 연산은 실행과정 동안 변하는 값이 아니기 때문에, 컴파일 시에 컴파일러가 계산해서 그 결과로 대체함으로써 코드를 보다 효율적으로 만듭니다. 컴파일러가 미리 덧셈 연산을 수행하기 때문에 실행 시에는 덧셈 연산이 수행되지 않습니다. 수식에 변수가 들어가있는 경우에는 컴파일러가 미리 계산을 할 수 없기 때문에 형 변환을 해줘야 합니다.
char ch2 = (char)(c1+1) 그렇지 않으면 컴파일 에러가 발생합니다.

float 타입의 값을 double 타입으로 형 변환하면, 부호와 지수는 달라지지 않고 그저 기수의 빈자리를 0으로 채울뿐이므로 0.1f를 double 타입으로 형 변환해도 그 값은 여전히 달라지지 않습니다. 즉, float 타입의 값을 정밀도가 더 높은 double 타입으로 형변환했다고 해서 오차가 적어지는 것이 아니라는 말입니다.

str1과 str2의 내용이 "abc"로 같은데도 ==로 비교하면 false의 결과를 얻게됩니다. 내용은 같지만 str1과 str2가 서로 다른 객체이기 때문입니다. 그러나 equals()는 객체가 달라도 내용이 같으면 true를 반환합니다. 그래서 문자열을 비교할 때는 항상 equals()를 사용해야 합니다.

  • 효율적인 연산
    • OR 연산 '||'의 경우, 두 피연산자 중 어느 한 쪽만 '참’이어도 전체 연산결과가 '참’이므로 좌측 피연산자가 'true(참)'이면, 우측 피연산자의 값은 평가하지 않습니다.
    • AND 연산 '&&'의 경우, 어느 한 쪽만 '거짓(0)'이어도 전체 연산결과가 '거짓(0)'이므로 좌측 피연산자가 '거짓(0)'이면, 우측 피연산자의 값은 평가하지 않습니다.

비트 XOR 연산자 '^'는 두 피연산자의 비트가 다를 때만 1이 됩니다. 그리고 같은 값으로 두고 XOR 연산을 수행하면 원래의 값으로 돌아온다는 특징이 있어서 간단한 암호화에 사용됩니다.

비트 전환 연산자는 피연산자의 타입이 int보다 작으면 int로 자동 형변환(산술 변환) 후에 연산하기 때문에 연산 결과는 32자리의 2진수입니다.

쉬프트 연산자의 좌측 피연산자는 산술변환이 적용되어 int보다 작은 타입은 int 타입으로 자동 변환되고 연산결과 역시 int 타입이 됩니다. 그러나 쉬프트 연산자는 다른 이항연산자들과 달리 피연산자의 타입을 일치시킬 필요가 없기 때문에 우측 피연산자에는 산술변환이 적용되지 않습니다.

변수 앞에 키워드 final을 붙이면 상수가 됩니다. 상수는 반드시 선언과 동시에 값을 저장해야하며, 한 번 저장된 값은 바꿀 수 없습니다.
