이번에는 Java로 알고리즘을 공부하면서 입/출력을 받을 때 사용하는 함수에 대해서 공부해보았습니다.

Scanner

사용하기 편리하기 때문에 간단하게 코드를 짜는 경우는 Scanner를 쓰는 것이 좋아보입니다. 그러나 Scanner로 입력 받는 데에 한계를 느낄 때 사용하는 것이 BufferedReader입니다.

BufferedReader

사용방식

선언하는 부분에만 유의하면 되고, 다른 점은 데이터를 readLine()함수를 이용해서 읽어올 경우 String 리턴값을 가진다는 점입니다. 그렇기 때문에 parseInt()` 함수를 이용하여 String을 int형으로 파싱해주었습니다.

Scanner의 경우 그냥 nextInt를 입력하면 알아서 공백 다음 숫자를 받기 때문에 어려움 없이 사용할 수 있지만, 이 부분에서도 BufferedReader를 사용할 경우에는 조금 다른 방식을 취해야합니다. StringTokenizer를 이용하는 방식입니다. 이름과 같이 String을 특정 구분자값을 통해 분리하는 함수입니다. 기본적으로는 공백을 기준으로 문자열을 가릅니다. split()함수와 비슷하다고 생각하면 됩니다.

StringTokenizer

선언한 StringTokenizer에 nextToken() 함수를 이용하면 br.readLine()을 통해 입력받은 값을 공백단위로 구분하여 nextToken()을 통해 순서대로 호출할 수 있습니다. 기본적으로는 공백이 구분자로 사용되지만, StringTokenizer 선언시 2번째 파라미터로 별도로 구분자를 추가할 수 있습니다.

BufferedWriter

일반적으로는 System.out.println() 함수를 많이 사용합니다. 사실 해당 부분은 테스트 케이스가 작을 때에는 크게 속도 차이가 나지 않는다고 알고 있습니다. 그러나 테스트 케이스가 무척 커지면 이러한 사소한 부분에서도 실행 가능하고 불가능한 여부가 결정되기 때문에 알고리즘 문제를 풀 때에 BufferedWriter를 사용하는 편이 좋을 것입니다.

System.out.println()과 동일하게 사용하면 됩니다. 하지만, 주의할 점은 마지막에 flush() 함수나 close() 함수를 쓰지 않은 경우 제대로 출력되지 않을 수 있다는 점입니다.
