Firebase란??

위의 방식과 같이 loginId() 함수안에서 task가 isSuccessful일 때 HomeActivity로 넘어가게 된다면 구조는 아래와 같은 구조가 되어버립니다.

Email Login만 사용할 경우에는 크게 문제가 되지 않습니다. 하지만, FaceBook, Google 등의 소셜 로그인을 같이 사용하게 되면 FaceBook 로그인이 되어 있는데, Google 로그인을 요청하는 등의 뒤죽박죽 되는 상황이 발생하게 됩니다.

그래서 AuthStateListener을 통해서 통합으로 세션을 관리해줘서 다음 화면으로 넘어갈 수 있게끔 도와주는 것입니다. 그래서 HomeActivity로 넘어가는 코드는 저 위치가 아니라 authStateListener에 위치하는 것이 적당하다고 생각합니다.

Firebase를 사용하여 Google 로그인을 할 때 발생하는 문제 중 하나는 로그인에 성공하면 Activity가 중복으로 실행되어서 쌓이게 되는 문제입니다. 이 문제는 onStop()이 실행되지 않아서 발생하는 버그입니다.

원래는 구글 로그인을 할 때 잠깐 authStateListener를 떼었다가 끝나고 돌아올 때 다시 붙여야 되는데, onStop()이 실행되지 않아서 붙이기만 해서 중복 Acitivity가 실행되게 되는 것입니다.

그래서 onStop() -> onPause()로 바꿔주게 되면 정상적으로 실행이 됩니다.

데이터 읽기(동기)

실시간으로 데이터를 전송받는 방식 : RTSP(Real Time Streaming Protocol)

이 기능은 SnapShot 이라는 코드를 사용하고 주로 채팅이나 좋아요를 구현할 때 사용합니다. 실시간으로 서버와 통신하기 위해서는 소켓이라는 모듈을 설치해서 복잡하게 코드를 작성해야 했으나, RTSP를 사용하면 코드 몇줄만으로 내가 원하는 앱을 쉽게 만들 수 있습니다.

스냅샷이 시스템 리소스를 많이 차지합니다. 만약 리스트가 1000줄 정도 된다고 가정하면 맨 위의 데이터를 지웠을 경우, 맨 위의 데이터만 지우는 것이 아니라 1000개 중에 지운 내용 1개 데이터를 제외한 나머지 999개를 다시 다운로드 받아서 다시 그리는 것과 같이 비효율적인 구조입니다. 이것을 방지하고자 하는 코드가 있습니다. 아래와 같습니다.
