LinearLayout

  1. 특징
  • 레이아웃이 일렬로 들어감
  • 구현하기 쉽고 많이 쓰임
  • 비율로 화면을 구성할 수 있음
  1. 속성
  • Weight : 화면을 비율로 나눌 때 즉, 화면 상에서 얼마만큼의 비율을 차지할 것인가를 결정할 수 있음
  • Gravity : LinearLayout 안에 있는 뷰를 정렬시킬 때 사용함 즉, 안에 있는 뷰를 오른쪽,왼쪽, 가운데 정렬 등등을 할 때 사용함
  • Orientation : LinearLayout에만 있는 속성으로 방향성을 설정해줌

RelativeLayout

  1. 특징
  • 상대적인 레이아웃으로 다른 상대의 위치에 따라 나의 위치가 정해진다고 생각하면 된다.
  • 구현하기 어렵고 복잡
  • 디자인 구성이 다양한 장점
  1. 속성
  • Alignparent : 부모 뷰와 나란히 하겠다는 의미
  • Align : 타겟된 뷰와 나란히 하겠다는 의미
  • To(방향)of, below/above : 타겟된 뷰의 왼쪽, 오른쪽, 위, 아래로 뷰를 위치시킬 수 있다. 중요한 속성

ConstraintLayout

  1. 특징
  • LinearLayout, RelativeLayout의 특성을 가짐
  • UI 툴로 인한 사용법이 편리
  • 다양한 화면에서 구현이 가능 즉, 하나의 레이아웃으로 스마트폰이나 태블릿 PC의 레이아웃을 구성할 수 있다.
  • 아직까지는 베타버전 느낌이 강해서 레이아웃을 만들 때 종종 멈추는 버그가 있지만, 추후에 패치되면서 안정화가 된다면 많이 사용할 수 있을 것 같다.
  1. 속성

  • autoConnect : 레이아웃 생성시 마다 자동적으로 Constraint 생성 설정

  • Clear All Constraints : 화면 전체 Constraint 지우기

  • Infer Constraint : Constraint 자동 지정하기 autoConnect가 꺼졌을 때 내가 지정한 뷰만 Constraint를 지정하고 싶을 때 사용한다. 하지만, 추론적으로 적용되기 때문에 내가 원하지 않은 방향으로 Constraint가 잡힐수도 있다.

  • Pack : 두 개 이상의 레이아웃을 이동하거나 채우고 연결할 때 사용

  • Align : 레이아웃 Align 시키기 (기준점은 큰 레이아웃이 된다.)

  • Guideline : 가이드라인 생성

  • Attribute : 레이아웃 특성 설정

  • View All Attribute : Attribute 상세

  • Edit BaseLine : 레이아웃 안에 있는 글자 라인 맞추기(자주 사용하는 기능)

  • Chain : 레이아웃을 그룹으로 만들어 주는 기능(자주 사용하는 기능) 그룹으로 묶고 싶은 뷰를 모두 선택한 후에
    – Create Horizontal Chain : 가로로 있는 뷰들을 그룹으로 묶어줌
    – Create Vertical Chain : 세로로 있는 뷰들을 그룹으로 묶어줌

  • 화면이 바뀌거나 디바이스가 바뀌어도 그에 대해 유동적으로 대응하기 위해서 Constraint와 Chain을 사용한다.

ConstraintLayout은 SOPT에서도 그렇고 많이 알려주지 않고 사용하지도 않아서 모르고 있었다. 하지만, 내가 생각했던 것보다 기능이 많았고, 앞으로 안정화된다면 더 공부해서 많이 사용해봐야겠다. 충분한 장점이 있을 것 같다고 생각한다. ^^

ImageView

  1. 속성

scaleType : 이미지 뷰의 크기에 따라서 원본 이미지를 조정하는 기능

  • Metric : 현재 사이즈 그대로 왼쪽 상단부터 채우면서 이미지를 표현. 나머지는 짤림
  • fitXY : 이미지를 이미지뷰에 맞게 높이와 넓이를 맞춤. 이미지의 크기가 작거나 큰 경우에도 이미지뷰에 크기에 맞게 높이와 폭이 맞춰진다.
  • fitStart : 이미지를 이미지뷰의 폭을 맞춘 후 상단에 배치시킨다.
  • fitCenter : 이미지를 이미지뷰의 폭을 맞춘 후 중앙에 배치시킨다.
  • fitEnd : 이미지를 이미지뷰의 폭을 맞춘 후 하단에 배치시킨다.
  • center : 이미지 사이즈를 그대로 유지한채 이미지를 가운데로 배치시키는 것. 만약 이미지가 이미지뷰보다 사이즈가 클 경우 나머지 부분은 삭제된다. 반대로 이미지가 이미지뷰보다 사이즈가 작은 경우에는 이미지가 가운데 배치되고 나머지 부분은 ImageView 바탕색이 채워진다.
  • CenterCrop : 이미지 높이를 이미지 뷰에 맞춘 후 중앙으로 옮긴다. 단, 이미지의 폭은 늘리지 않는다.
  • CenterInside : 이미지가 이미지 뷰보다 클 경우 fitCenter 작을 경우 Center 적용하는 것이 효과적(작을 경우에 이미지는 커지지 않고 그대로 중앙에 배치된다.)

안드로이드에서 이미지를 사용할 때는 300KB를 넘지 않는 것을 사용해야 한다. 왜냐하면 컴퓨터의 성능이 낮아지면 애뮬레이터의 성능도 낮아지기 때문에 메모리 부족으로 인해 App이 Crash 될 수 있기 때문이다. 300KB 이상의 이미지는 어떻게 가져오는지에 대해서는 추후에 안드로이드에 대해 파헤쳐보기라는 catergory에서 공부하면서 포스팅할 계획이기 때문에 나중에 확인할 수 있다.

기능

아래에 나오는 개념은 말로 표현하기에는 제약이 있어서 강의를 듣고 코드 실습 위주로 공부를 하였습니다. 영상을 보고 작성한 코드는 하단에 Github 링크를 통해서 확인할 수 있습니다.

  1. ProgressBar
  • 확정성 : 진행중인 작업 상태를 수치적으로 정확히 표현하거나 양, 갯수, 퍼센트를 보여주기 위해 사용
  • 불확정성 : 작업의 진행상태의 완료 시점이 확정되지 않은 경우 사용
  1. SeekBar
  • ProgressBar에 드래그할 수 있는 타겟을 추가하여 만든 객체
  • 휴대폰 밝기나 음량 크기를 조절할 때 많이 사용한다.
  1. SurfaceView

  • 코드를 작성하는 사람이 커스터마이징 할 수 있는 애니메이션 뷰라고 할 수 있다.
    움직이는 화면이나 카메라, 비디오 게임 등이 있으면 SurfaceView를 통해서 표현할 수 있다.

  • 즉 영상을 재생하고자 하는데, 뒤로가기, 재생, 정지 등의 버튼을 커스텀해서 사용하고 싶다면 SurfaceView에 영상 재생을 하고 버튼을 커스텀해서 이벤트 처리를 하면 된다.

  1. Dialog
  • 앱을 사용하다보면 간단한 메시지를 표현하기 위해 사용하는 것이다.
  1. Date & TimePicker
  • 날짜나 시간을 선택하기 위해서 기본적으로 제공하는 DatePicker, TimePicker를 사용할 수 있다.
  1. WebView
  • 안드로이드에 내장되 Web browser component로 안드로이드 앱 안에 HTML 코드를 실행시켜주는 View라고 볼 수 있다.
