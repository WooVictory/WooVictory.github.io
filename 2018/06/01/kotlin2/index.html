Class의 기본구조

  • Class는 Object라고도 한다.
  • 클래스는 크게 두 가지로 구성되어 있다.
  1. 변수들이 집합된 Property 부분
  2. 변수들을 가공시켜주는 Functions 부분
  • $ 기호를 사용하여 변수를 바로 참조할 수 있다.

Class - 상속

  1. 상속이란??
  • 즉 유전적인 성질을 물려받는다의 의미를 가지고 있다.
  • 상속을 하기 위해서는 허가를 주어야 한다.
  • 허가를 주는 방법은 선언자 앞에 open 명칭을 붙이면 된다.
  • 또한 변수나 함수에 open을 붙여주게 되면 상속을 받은 대상자가 클래스의 변수나 함수를 수정할 수 있는 권한을 가지게 된다.
  • 이 권한을 오버라이드라고 한다.
  1. Override(오버라이드)
  • 올라타다 의 의미를 가지고 있다.
  • 상속받은 변수나 함수를 덮어쓰는 개념이다. 즉, 사용하고 싶은 용도에 맞게 재정의해서 사용할 수 있는 것이다.
  • 상속을 통해서 위의 Class의 기본구조에서 연습했던 Human 클래스를 상속 받았다. 그리고 Human 클래스의 country와 name을 override해서 재정의하여서 속성 값을 변경하였다. 그리고 남은 성별 Property와 printHumanInfo 라는 함수도 사용할 수 있고, 여기서는 그대로 사용하였다.

Class - 인터페이스

  1. 인터페이스란??
  • 서로 정보가 상호작용하면서 도와주는 매개체
  • 소통을 할 때 사용하는 매개체(정보를 교환할 때 도와주는 매개체)
  1. 선언방법

Class - 접근권한

  • 변수나 함수 또는 클래스를 만들게 되면 외부에서 접근해서 사용하는 경우가 많은데 그럴 경우 클래스를 보호하기 위해서 내부에서 접근 제한을 시킬 수 있다.
  1. private - 접근 범위 : 현재 클래스
  2. protected - 접근 범위 : 현재 클래스 + 상속 받은 클래스
    class는 protected가 존재하지 않음
  3. internal - 접근 범위 : 모듈
  4. public - 접근 범위 : 모든 범위
    클래스나 변수 앞에 아무것도 작성하지 않은 경우 default로 public이 설정된다.

DataClass

  1. 소개
  • 데이터 전용 클래스를 말한다. 기능이 없음
  • 코틀린에서는 데이터 클래스라는 기능을 도입하였고 생성자, getter/setter, Hashcode/equals, toString 과 같은 반복적인 작업들을 하지 않아도 기능들이 자동적으로 만들어지게 되었다.
  • 다른 클래스와 다르게 중괄호가 아닌 소괄호를 사용한다.
  1. 선언 방법

Nested Class

  1. Nested Class란 무엇인가??
  • 자주 쓰이는 개념
  • 중첩 클래스라는 것으로 클래스 안에 있는 클래스를 말한다.
  • 코드의 구조
  • 장점 : 추후에 수정과 변경이 용이
  1. Nested Data Class
  • 코드의 구조
  1. Nested Class 연습
  1. Nested Data Class 연습
  • Data Class를 사용함으로써 훨씬 간결해 진 것을 확인할 수 있다.

Tip - 안드로이드 스튜디오 단축키

  1. Basic 단축키 : 코드가 생각나지 않았을 때 도움을 받을 수 있음 control + Space
  2. ReformatCode 단축키 : 코드가 뒤죽박죽 되어있을 때 정렬할 수 있음 Command + option(alt) + L
  3. 주석 설정 단축키 : Command + Shift + /
  4. Show Intent Actions : 기능이나 코드가 제대로 입력되지 않았을 때 발생하는 오류를 잡아줌 Option(Alt) + Enter ex) View.OnClickListenser를 상속받았을 때 함수가 제대로 만들어지지 않았을 때 onClikc 함수를 implements하는 과정
