AsyncTask란??

인프런 강의를 통해서 코틀린에 대한 개념 및 안드로이드 위젯 사용법을 복습하는 차원에서 공부하고 있었다. 순서대로 배우는 과정에서 ProgressBar라는 것을 배웠고, 그 중에서 확정성 ProgressBar를 사용하는 과정에서 등장하는 AsyncTask라는 개념이 등장하였지만 무엇인지 몰라서 찾아보기로 했다.

안드로이드에서 UI를 조작할 수 있는 방법은 두 가지가 있다고 한다,.

  1. Handler와 Looper 사용하기
  2. runOnUiThread 사용하기

위의 내용은 추후에 공부하면서 정리할 계획이다.

AsyncTask는 안드로이드에서 스레드나 메시지 루프 등의 원리를 이해하지 않아도 하나의 클래스에서 UI 작업을 쉽게 할 수 있게 해주는 클래스다. 이해하게 된다면 간단하고 쓰기 편하기 때문에 이것만 사용한다고 할 수도 있지만, 단점이 존재하기 때문에 주의해서 사용해야 한다고 한다.

AsyncTask의 개념

  • AsyncTask의 동작 순서를 먼저 살펴보고 개념을 보도록 하자
  1. execute() 명령어를 통해 AsyncTask를 실행한다.
  2. AsyncTask로 백그라운드 작업을 실행하기 전에 onPreExcuted() 가 실행된다. 이 부분에는 이미지 로딩 작업이라면 로딩 중 이미지를 띄워 놓기 등 스레드 작업 이전에 수행할 동작을 구현한다.
  3. 새로 만든 스레드에서 백그라운드 작업을 수행한다. execute() 메소드를 호출할 때 사용된 파라미터를 전달 받는다.
  4. doInBackground()에서 중간 중간 진행 상태를 UI에 업데이트 하도록 하려면 publishProgress() 메소드를 호출해야 한다.
  5. onProgressUpdate() 메소드는 publishProgress()가 호출 될 때마다 자동으로 호출된다.
  6. doInBackground() 메소드에서 작업이 끝나면 onPostExcuted()로 결과 파라미터를 리턴하면서 그 리턴값을 통해 스레드 작업이 끝났을 때의 동작을 구현한다.

여기서 핵심이 되는 것은 onPreExecute(), onProgressUpdate(), onPostExecute() 메소드는 Main 스레드에서 실행되므로 UI 객체에 자유롭게 접근할 수 있다는 것이다.

AsyncTask Generic 타입의 인자들이란??

AsyncTask <Params, Progress, Result>

Params : doInBackground 파라미터 타입이 되며, execute 메소드 인자 값이 된다.
Progress : doInBackground 작업 시 진행 단위의 타입으로 onProgressUpdate 파라미터 타입이다.
Result : doInBackground 리턴 값으로 onPostExecute 파라미터 타입이다.

  • AsyncTask의 제약조건 및 단점

AsyncTask는 비교적 오래 걸리지 않은 작업에 유용하고, Task 캔슬이 용이하며 로직과 UI 조작이 동시에 일어나야 할 때 매우 유용하게 사용되지만 다음의 제약조건과 단점을 알고 사용해야 한다.

제약조건

  • API16(젤리빈) 미만 버전에서는 AsyncTask 선언을 UI Thread에서 해주지 않으면 오류가 발생한다. (API 16 이상의 버전에서는 자유롭게 사용 가능)
  • excetes(Params)는 UI 스레드에서 직접 호출해야 한다.
  • 수동으로 onPreExecute(), onPostExecute(Result),doInBackground(Params…), onProgressUpdate(Progress…) 호출하면 안된다.
  • Task는 오직 한번만 실행될 수 있다.

단점

  • 하나의 객체이므로 재사용이 불가능하다. 그렇다고 객체를 새롭게 생성하면 된다고 생각할 수 있지만, 이런 경우에 메모리 효율이 나빠진다.
  • 구현한 액티비티 종료 시 별도의 지시가 없다면 종료되지 않는다.
  • Activity 종료 후 재시작 시 AsyncTask의 Reference는 invail해지며 onPostExecute() 메소드는 새로운 Activity에 어떠한 영향도 끼치지 못한다.
  • AsyncTask의 기본 처리 작업 개수는 1개이다. Async 하지 않은 AsyncTask의 변화는 아래와 같다.

AsyncTask는 순차적으로 하나의 스레드에 실행되었으나, 도넛의 시작과 함께 병렬 처리 가능한 스레드 풀로 바뀌었다고 한다. 허니콤의 시작과 함께 병렬 처리의 에러를 피하기 위해 다시 싱글 스레도로 실행되게 되었다. 만약 병렬 처리를 원한다면
myTask.executeOnExecutor(AsyncTask.THREAD_POOL_EXECUTOR) 처럼 사용할 수 있다.
